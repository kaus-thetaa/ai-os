# AI-OS
<img width="800" height="509" alt="Image" src="https://github.com/user-attachments/assets/75e266b6-182a-413b-a736-1ab6362ca31f" />
A desktop application that lets you interact with your Windows file system using
plain English — search, organize, rename, and manage files through a local AI
layer that runs entirely on your machine.

No cloud APIs. No subscriptions. No data ever leaves your device.

## Why offline matters

Most "AI file assistant" tools are thin wrappers around a cloud API — your file
names, folder structure, and sometimes file contents get sent to a third-party
server. AI-OS doesn't do this. Every feature — the language model, the
embeddings, the vector search — runs locally. You could disconnect your
internet entirely and AI-OS would work exactly the same.

## Features

- **Natural language file search** — search by meaning, not just keywords.
  Searching "tax documents" finds `2023_returns_final.pdf` even if the words
  don't literally match, using local embeddings and vector search.
- **Folder and file summarization** — get a plain English summary of what a
  file or an entire folder contains, generated by a local LLM.
- **Content-based file renaming** — files get renamed based on what they
  actually contain, not left as `Untitled document (3).txt`.
- **Automatic folder organization** — sort files into folders by type
  automatically, or describe what you want in plain English
  ("separate images and videos into two folders") and AI-OS plans and
  executes it.
- **Duplicate file detection** — finds files with identical content via
  hashing, and safely sends duplicates to the recycle bin (never a permanent
  delete).
- **Local knowledge base** — build a searchable index across PDFs, Word
  documents, and text files, all stored and queried locally.

## Tech stack

- **Python 3.13**
- **Ollama** running **Phi-3 Mini** — local LLM inference, no cloud calls
- **PyQt6** — desktop UI
- **sentence-transformers** — offline embeddings (`all-MiniLM-L6-v2`)
- **ChromaDB** — local vector database for semantic search
- **pypdf** / **python-docx** — local document text extraction
- **send2trash** — safe, recoverable file deletion

## Architecture

AI-OS is built as a set of independent, testable modules rather than one large
application file:
config.py              → shared paths, settings, and constants
llm_engine.py           → wrapper around Ollama / Phi-3 Mini
embeddings_engine.py     → text-to-vector conversion via sentence-transformers
vector_store.py           → ChromaDB persistence and semantic search
file_scanner.py             → file system reading, metadata, classification
search_engine.py              → natural language search (text files)
summarizer.py                   → LLM-powered file and folder summaries
renamer.py                        → content-based renaming with safety checks
organizer.py                        → folder organization by type or custom rule
duplicate_finder.py                   → hash-based duplicate detection
knowledge_base.py                       → pdf/docx aware indexing
command_interpreter.py                    → free text → structured file actions
ui_main_window.py       → main window, sidebar navigation
ui_search_panel.py        → search interface
ui_summarize_panel.py        → summarization interface
ui_rename_panel.py              → rename with review-before-apply
ui_organize_panel.py               → organize with plan-before-execute
ui_duplicates_panel.py                → duplicate review and safe deletion
ui_knowledge_base_panel.py               → pdf/docx indexing and search
styles.py                                   → centralized minimal theme
main.py                                        → application entry point


Every backend module is independently testable from the command line before
any UI depends on it — each file includes its own self-test block.

## Safety principles

Since this application performs real operations on real files, every
destructive or file-altering action follows explicit safety rules:

- **Never overwrite an existing file.** Renaming and organizing both check
  for name collisions and skip rather than silently overwrite.
- **Never permanently delete.** Duplicate removal uses the Windows recycle
  bin (`send2trash`), so any mistake is recoverable.
- **Plan before execute.** Free-text commands (via the command interpreter)
  and content-based renaming both show the proposed action before anything
  touches disk — nothing runs automatically without confirmation.

## Setup

### Requirements

- Windows 10/11
- Python 3.13+
- [Ollama](https://ollama.com/download) installed and running

### Installation

```powershell
git clone https://github.com/kaus-thetaa/ai-os.git
cd ai-os

python -m venv venv
venv\Scripts\activate

pip install -r requirements.txt

ollama pull phi3
```

### Running

```powershell
python main.py
```
