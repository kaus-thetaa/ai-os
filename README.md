# AI-OS

<img width="800" height="509" alt="AI-OS screenshot" src="https://github.com/user-attachments/assets/75e266b6-182a-413b-a736-1ab6362ca31f" />

A desktop application that lets you interact with your Windows file system using plain English — search, organize, rename, and manage files through a local AI layer that runs entirely on your machine.

No cloud APIs. No subscriptions. No data ever leaves your device.

## Why offline matters

Most "AI file assistant" tools are thin wrappers around a cloud API — your file names, folder structure, and sometimes file contents get sent to a third-party server. AI-OS doesn't do this. Every feature, the language model, the embeddings, the vector search, runs locally. You could disconnect your internet entirely and AI-OS would work exactly the same.

## Features

**Natural language file search**
Search by meaning, not just keywords. Searching "tax documents" finds `2023_returns_final.pdf` even if the words don't literally match, using local embeddings and vector search.

**Folder and file summarization**
Get a plain English summary of what a file or an entire folder contains, generated by a local LLM.

**Content-based file renaming**
Files get renamed based on what they actually contain, not left as `Untitled document (3).txt`.

**Automatic folder organization**
Sort files into folders by type automatically, or describe what you want in plain English, like "separate images and videos into two folders," and AI-OS plans and executes it.

**Duplicate file detection**
Finds files with identical content via hashing, and safely sends duplicates to the recycle bin, never a permanent delete.

**Local knowledge base**
Build a searchable index across PDFs, Word documents, and text files, all stored and queried locally.

## Tech stack

Python 3.13

Ollama running Phi-3 Mini for local LLM inference, no cloud calls

PyQt6 for the desktop UI

sentence-transformers for offline embeddings (`all-MiniLM-L6-v2`)

ChromaDB as the local vector database for semantic search

pypdf and python-docx for local document text extraction

send2trash for safe, recoverable file deletion

## Architecture

AI-OS is built as a set of independent, testable modules rather than one large application file.

**Backend**

| File | Purpose |
|---|---|
| `config.py` | shared paths, settings, and constants |
| `llm_engine.py` | wrapper around Ollama / Phi-3 Mini |
| `embeddings_engine.py` | text-to-vector conversion via sentence-transformers |
| `vector_store.py` | ChromaDB persistence and semantic search |
| `file_scanner.py` | file system reading, metadata, classification |
| `search_engine.py` | natural language search (text files) |
| `summarizer.py` | LLM-powered file and folder summaries |
| `renamer.py` | content-based renaming with safety checks |
| `organizer.py` | folder organization by type or custom rule |
| `duplicate_finder.py` | hash-based duplicate detection |
| `knowledge_base.py` | pdf/docx aware indexing |
| `command_interpreter.py` | free text to structured file actions |

**Interface**

| File | Purpose |
|---|---|
| `ui_main_window.py` | main window, sidebar navigation |
| `ui_search_panel.py` | search interface |
| `ui_summarize_panel.py` | summarization interface |
| `ui_rename_panel.py` | rename with review-before-apply |
| `ui_organize_panel.py` | organize with plan-before-execute |
| `ui_duplicates_panel.py` | duplicate review and safe deletion |
| `ui_knowledge_base_panel.py` | pdf/docx indexing and search |
| `styles.py` | centralized minimal theme |
| `main.py` | application entry point |

Every backend module is independently testable from the command line before any UI depends on it. Each file includes its own self-test block.

## Safety principles

Since this application performs real operations on real files, every destructive or file-altering action follows explicit safety rules.

Renaming and organizing never overwrite an existing file. Both check for name collisions and skip rather than silently overwrite.

Duplicate removal never permanently deletes. It uses the Windows recycle bin via `send2trash`, so any mistake is recoverable.

Free-text commands and content-based renaming always plan before they execute. The proposed action is shown before anything touches disk, nothing runs automatically without confirmation.

## Setup

Requirements: Windows 10/11, Python 3.13+, and [Ollama](https://ollama.com/download) installed and running.

```powershell
git clone https://github.com/kaus-thetaa/ai-os.git
cd ai-os

python -m venv venv
venv\Scripts\activate

pip install -r requirements.txt

ollama pull phi3
```

Run it with:

```powershell
python main.py
```
